---
description: TDD with automated refactors
---

Perform Test-Driven Development with short cycles:
1. RED. Write a test. One test case. Run it, it should fail.
2. Implement the functionality. Re-run tests until they pass. GREEN.
3. REFACTOR. Review and refine your code or adjacent code if needed.
    1. Run automated linters and formatters with automated fixes.
    2. Run automated static type checking and fix issues.
    3. Review and refine the tests against test-desiderata skill.
    4. Review and refine your code against python code style skill.
4. Repeat.
